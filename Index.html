Test Description

This page attempts to trigger the type confusion vulnerability (CVE-2023-32439) in WebKit's JavaScript engine.

The vulnerability involves incorrect type handling in the DFG JIT compiler during optimization phases, specifically affecting how the EnumeratorNextUpdateIndexAndMode and HasIndexedProperty DFG nodes are processed :cite[2].

Progress: Not started

Technical Details

The vulnerability occurs due to:

  • Incorrect equivalence handling between EnumeratorNextUpdateIndexAndMode and HasIndexedProperty DFG nodes during the Common Subexpression Elimination (CSE) phase
  • Type confusion between SpecBytecodeNumber (EnumeratorNextUpdateIndexAndMode) and SpecBoolean (HasIndexedProperty)
  • This allows the replacement of HasIndexedProperty with EnumeratorNextUpdateIndexAndMode during JIT compilation
// Simplified vulnerable code path:
for (let _ in arr) { // Creates EnumeratorNextUpdateIndexAndMode DFG node
    return 0 in arr; // Creates HasIndexedProperty DFG node
}

Protection Status

This vulnerability was patched in:

  • iOS 16.5.1 and iPadOS 16.5.1
  • iOS 15.7.7 and iPadOS 15.7.7
  • macOS Ventura 13.4.1
  • Safari 16.5.1

Ensure your device is updated to these or later versions to be protected :cite[4]:cite[7].
